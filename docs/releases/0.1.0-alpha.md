## Install

### One-liner (GitHub Release assets published)

```bash
curl -fsSL https://aura.fadosoft.com/install.sh | bash
# pin:    curl -fsSL https://aura.fadosoft.com/install.sh | AURA_VERSION=0.1.0-alpha bash
# switch: avm --list && avm 0.1.0-alpha
```

Installs under **`~/.aura/versions/<ver>/`** with `current` + `bin/aura` (version-switch ready). Release assets: [v0.1.0-alpha on GitHub](https://github.com/auraspace/aura/releases/tag/v0.1.0-alpha).

### From a clone (always works)

```bash
git clone https://github.com/auraspace/aura.git
cd aura
cargo install --path crates/aura-cli
aura version
aura new hello && aura run hello
```

`aura_rt.c` is **embedded** in the CLI. After install you do **not** need the monorepo on disk for `build` / `run` / `test`. Override with `AURA_RUNTIME=/path/to/aura_rt.c` if needed.

### From the monorepo without installing

```bash
cargo run -p aura-cli -- run examples/notes
```

### Prerequisites

- Rust stable (`cargo`, `rustc`)
- System C compiler (`cc` / `clang` / `gcc`) on `PATH`
- Optional: `pnpm` for the docs site only

### Release tarball (when published)

## In scope (alpha)

| Area         | What works                                                                                                                                                                                                                                                          |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **CLI**      | `new`, `init`, `version`, `check`, `build`, `run`, `test`, `emit-c`                                                                                                                                                                                                 |
| **Language** | classes, interfaces, generics+bounds, struct, enum/`match`/`Result`, exceptions, packages/`import`, Array, null-safety (`?`/`!!`/`?:`/`?.`), `if` expr, lambdas/fun types (val Int/Bool/String capture), `is`, type alias, `const`, string `+` / interp / substring |
| **Runtime**  | GC mark/sweep (class heap), exceptions, Array ownership, Fun env free                                                                                                                                                                                               |
| **Std**      | `std.io` (console + file), `std.assert`, `std.collections` (Map/Set/HashMap/Iterable + Int HOF)                                                                                                                                                                     |
| **Packages** | Path deps + `aura.lock` (registry schema only, no fetch)                                                                                                                                                                                                            |
| **Dogfood**  | `examples/notes`                                                                                                                                                                                                                                                    |

## Out of scope (explicit)

- `async` / `await` / tasks / channels
- Macros / plugins / reflection
- LLVM backend (C backend only)
- Package **registry** fetch / semver resolve
- True `ref` / borrow types; `Array` of interface elements
- Generic `HashMap<K,V>` (Stringâ†’Int only)
- Lambda capture of class / Array / `var`
- CLI argv / stdin line reader
- Windows CI matrix as a release gate (best-effort later)
- Signed / notarized installers

See [`agents/debts.md`](../../agents/debts.md) for open technical debt.

## Known limitations

1. **Strings** are C strings: no embedded NUL; indices are UTF-8 **bytes**.
2. **File I/O** throws `String` messages (not `Result` yet); max read size 256 MiB.
3. **No program argv** â€” demos use fixed paths/scenarios.
4. **Path deps only** for multi-package work in the monorepo.
5. App binaries need a C toolchain on the **machine that runs `aura build`**, not on end users of the produced binary.
