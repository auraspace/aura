## Why this batch

`0.1.0-alpha` freezes install + language MVP, but release notes still block real tools:

- no program **argv** / **stdin**
- lambda capture limited to immutable `val` Int/Bool/String
- String surface thin for parsing (`substring` only)
- HashMap still String→Int only

This batch closes those gaps **without** async, registry HTTP, LLVM, true borrow, or Array-of-interface.

## Design notes (lock before coding)

### Args

- Prefer **`std.io.args(): Array<String>`** over changing `fun main()` signature (keeps all existing corpus green).
- `args()[0]` = program name (C `argv[0]`); user flags from index 1 — document like most systems.
- Codegen: generated `main` stores argc/argv into runtime globals before calling user `main`.

### Stdin

- `readLine(): String?` — line without trailing `\n`; `null` on EOF; empty line is `Some("")` via empty string + non-null (String is non-null; use `String?` only for EOF).
- Max line size: match file I/O spirit (cap + throw or truncate — **cap + throw String** for MVP).

### Lambda captures

| Capture                  | Rule (MVP)                                            |
| ------------------------ | ----------------------------------------------------- |
| `val` class              | Copy GC ptr into env; env mark walks ptr slots        |
| `val` Array              | Store `{data,len,cap}` **view** (non-owning); no free |
| `var` Int/Bool           | Heap box; lambda reads/writes through box             |
| `var` class/Array/String | **Reject** this batch (debt)                          |
| Fun capture              | **Reject** this batch (nested env / double free risk) |

Env free (C11b) stays: free env struct on Fun drop; **do not** free Array buffers owned only by outer scope.

### HashMap String→String

- **Done (C12n):** `HashMapStr` open-addressing table (same `hash_string` + load ≥ 1/2 grow as `HashMap`); values `String`; `get` → `String?`.
- Factory `hash_map_str()` capacity 16; existing String→Int `HashMap` / `hash_map()` unchanged.
- No generic `HashMap<K,V>` yet.

### String HOF helpers

- **Done (C12o):** `map_strings` / `filter_strings` / `fold_strings` free functions in `std.collections` (mirror C10i Int helpers).
- Corpus: `std_collections/hof_str`. Generic map/filter still deferred.

### Soft file read

- **Done (C12p):** `std.io.tryReadFile(path): String?` — null on missing path / I/O / oversize / empty path / OOM / embedded NUL; throwing `readFile` unchanged.
- Runtime `aura_try_read_file`; corpus `std_io/try_read_file`.

### Dogfood CLI

- **Done (C12q):** `examples/wc` — `args()` flags/path, `tryReadFile`, `split`/`trim`/`indexOf`/`toInt`/`startsWith`, `exit`; `aura run examples/wc -- <file>`; `@test` helpers. Prefer focused `wc` over extending notes (notes stays fixed-scenario).

### Out of scope (this batch)

- `async` / tasks / channels
- Macros, reflection, LLVM backend
- Registry HTTP client / semver resolve (schema stays C8k)
- True `ref` / borrow types; Array field return still moves
- `Array` of interface elements
- Generic `HashMap<K,V>` + hashable protocol
- Fun-in-env capture; shared Fun copy GC for env
- Signed / notarized installers
- Changing default `main` arity
