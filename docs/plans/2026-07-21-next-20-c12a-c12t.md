## Design notes (lock before coding)

### Args

- Prefer **`std.io.args(): Array<String>`** over changing `fun main()` signature (keeps all existing corpus green).
- `args()[0]` = program name (C `argv[0]`); user flags from index 1 ŌĆö document like most systems.
- Codegen: generated `main` stores argc/argv into runtime globals before calling user `main`.

### Stdin

- `readLine(): String?` ŌĆö line without trailing `\n`; `null` on EOF; empty line is `Some("")` via empty string + non-null (String is non-null; use `String?` only for EOF).
- Max line size: match file I/O spirit (cap + throw or truncate ŌĆö **cap + throw String** for MVP).

### Lambda captures

| Capture                  | Rule (MVP)                                            |
| ------------------------ | ----------------------------------------------------- |
| `val` class              | Copy GC ptr into env; env mark walks ptr slots        |
| `val` Array              | Store `{data,len,cap}` **view** (non-owning); no free |
| `var` Int/Bool           | Heap box; lambda reads/writes through box             |
| `var` class/Array/String | **Reject** this batch (debt)                          |
| Fun capture              | **Reject** this batch (nested env / double free risk) |

Env free (C11b) stays: free env struct on Fun drop; **do not** free Array buffers owned only by outer scope.

### HashMap StringŌåÆString

- Duplicate open-addressing table with `char *` values (or length-prefixed); no generic `HashMap<K,V>` yet.
- Keep existing StringŌåÆInt API; add `hash_map_str()` / type name TBD in std.collections.

### Out of scope (this batch)

- `async` / tasks / channels
- Macros, reflection, LLVM backend
- Registry HTTP client / semver resolve (schema stays C8k)
- True `ref` / borrow types; Array field return still moves
- `Array` of interface elements
- Generic `HashMap<K,V>` + hashable protocol
- Fun-in-env capture; shared Fun copy GC for env
- Signed / notarized installers
- Changing default `main` arity
