## Why this batch

C12 closed alpha gaps for CLI programs, but open debt still blocks polish and the next product step (third-party packages):

- method call on temporary (`arr.get(i).trim()`) fails codegen
- no `Int.toString` / numeric interpolation
- owned `String` segments from `split`/`trim*` leak when Array drops
- Fun-in-env capture still rejected (nested closures)
- registry: lock schema only (C8k); no fetch/semver (RFC-005 K1)

This batch addresses those **without** async, LLVM, true borrow, `Array<I>`, or `aura publish`.

## Design notes (lock before coding)

### C13b — method on temporary

- When method recv is a **call expression** (or other rvalue) with type that needs addressable storage for C (`const char *` methods, etc.), emit:

  ```c
  T _aura_tmp_N = <recv expr>;
  method(&_aura_tmp_N, ...);  // or pass by value if API is by value
  ```

- Priority types: **String** methods. Array methods if free.
- Do not invent full temporary SSA — local counter in codegen ctx is enough.
- Exit: corpus `x.get(i).trim()` and `.toInt()` compile and run.

### C13c — Int.toString

- Prefer builtin method on `Int` → `String` (decimal, no locale).
- Runtime: `char *aura_i64_to_string(int64_t)` malloc’d; document ownership = caller/Array rules (same as other owned strings).
- Optional same slice: interpolation allows Int (desugar to `toString` + `+`).
- Exit: print counts without local `u64ToString` hack in examples.

### C13d — Array\<String\> free

- On drop/clear/set of Array whose elem is String (C `const char *` owned copies from split/trim/substring/+):
  - free each `data[i]` if non-null, then free buffer (existing).
- Nested `Array<Array<String>>` should free inner via existing nested Array free **plus** string elems.
- Do **not** free string literals / non-owned views — if MVP cannot distinguish, document: only free elems of arrays produced by known owning ops; or tag ownership later. **Preferred MVP:** free all `const char *` elems on Array\<String\> drop (literals should not be stored in owned arrays without strdup — verify codegen for `["a"]` literals).
- Exit: corpus that split → drop does not leak under ASan **if** available; else logical free path review + unit-style run.

### C13e — Fun capture

- Allow capturing outer `val` Fun (fat pointer `{env,fn}`) into new env.
- Mark: if env is GC-managed or registered, walk Fun slots and mark nested env.
- Drop: release nested env once (refcount **or** single-owner move). **Must not double-free.** Prefer retain/release on env or move-only capture of Fun (document).
- Reject recursive escape patterns that MVP cannot handle with a clear diagnostic if needed.
- Exit: corpus nested `map` style Fun-in-Fun.

### C13f — var String capture

- Mirror C12m boxes: `aura_box_str` with refcount; mutate through box.
- Exit: two lambdas share `var s: String` mutations.

### C13h — diagnostics

- Sema: unsupported capture → single clear error with supported list.
- No codegen change required.

### C13i–l — registry K1 (GitHub-backed, RFC-005)

| Step   | Behavior                                                                                                  |
| ------ | --------------------------------------------------------------------------------------------------------- |
| Index  | Read `packages/<name>/versions.json` from fixture dir or cache; schema per RFC-005                        |
| Semver | Default caret on `"1.2"`; pin exact in lock `version` + `checksum` + `source`                             |
| Fetch  | HTTP optional; **tests use `file://` or pre-seeded cache** so CI offline-green                            |
| Build  | Path deps unchanged; registry pins resolve to cache src dir on `AURA_REGISTRY_CACHE` / `~/.aura/registry` |

**Non-goals K1:** `aura publish`, live GitHub rate-limit polish, workspaces, multiple registries UI.

### C13m — String ASCII

- Pick **either** `replace(old, new): String` **or** `toLower()`/`toUpper()` ASCII; document UTF-8 byte caveat.
- Do not expand full Unicode.

### C13n–o — std.io

- `eprintln(s: String)` → stderr + newline.
- `tryWriteFile`: false/null on failure; keep throwing `writeFile`.

### C13p — Hashable spike

- Written design only unless a second concrete map is trivial.
- Record next step for C14.

### C13p shipped notes

- Design note: [2026-07-21-c13p-hashable-spike.md](./2026-07-21-c13p-hashable-spike.md)
- Decision: keep concrete `HashMap` / `HashMapStr`; C14 = monomorphized `HashMap<K,V>` + `Hashable` interface bound (not erase).

### C13q — dogfood

- Prefer fix `examples/wc` to use method-on-temp + `toString`.
- Optional: tiny package depending on fixture registry after C13l.
