## Design notes

### C6a — deep mark

Runtime only (no type maps yet): each `aura_gc_alloc` records `size`. Mark from roots, then **conservatively scan** each marked object's bytes as pointer-sized slots; any slot that is a known GC object is marked and enqueued. Cycles OK via mark bit.

### C6b — Array params

- Function/method `Array` params are **owners** (free at scope end unless returned by name).
- Call site: if arg is owner `Ident`, **move** after call (zero source, unmark).
- Non-owner args still shallow-copy while param still frees — prefer only pass owners/`Array(...)` until C6i.

### C6c — Iterable

No generic interfaces yet. Formal protocol shape on **interface** types:

- `fun len(): Int`
- `fun get(i: Int): E` → element type `E`

`for-in` on `Ty::Interface` uses iface dispatch. Class duck (`len` field or method + `get`) remains for C4y.
