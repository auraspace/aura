## Overview

Ship ten P3 slices after **C3z**. Order prefers: close class/GC debts → package interfaces → string/Array hygiene → stdlib DX → sema safety → lock graph. Stay on C backend; no LLVM/registry server.

## Architecture decisions

1. **Class `==` is identity** (pointer) — RFC-003; already works after C3y; ship corpus + docs (C4a).
2. **`Class?` is the same C pointer type as `Class`** — null is `NULL`; fix nullable emit (C4b).
3. **`Array<Class>` stores heap refs** — element type is class pointer; no deep free of elements (GC owns them).
4. **Interfaces get package keys** like classes (C3v pattern).
5. **String `==` is content** (`strcmp`); class stays identity.
6. **Structs/enums: no `==` in MVP** — reject in sema (C cannot compare aggregates).
7. **Auto-prelude** injects `std.io` for package builds only; single-file builtins remain.
8. **Registry / full Iterable** still deferred.
